# 订阅追踪功能实施路线图 v2

本路线图取代旧版七阶段执行顺序，但不删除旧文档。新版按技术边界合并为五个阶段，并允许“自动加入中文搜索”先于数据库阶段完成。

## 当前项目状态

- 工作区已有“自动加入中文搜索”的未提交实现和测试雏形。
- 这批改动不涉及主数据库 schema，因此可以保留，不需要为了阶段编号回退。
- 正式继续前必须先审计现有改动，确认其覆盖入口、请求取消和查询签名符合 [02-auto-chinese-search.md](02-auto-chinese-search.md)。
- 工作区不是干净基线。禁止执行会丢失现有改动的 reset、checkout 或全量代码生成。

## 新阶段与依赖

```text
02 自动加入中文搜索（可先实现，无数据库依赖）
 │
 ├──────────────┐
 │              v
01 数据基础 ──> 03 订阅追踪核心（缓存、高亮、聚合更新、计数）
                   │

                   │
```

推荐针对当前工作区的实际顺序：

1. 审计并完成阶段 02，保留现有实现。
2. 实施阶段 01；不得覆盖阶段 02。
3. 做一次 01+02 集成门禁，确认查询签名与数据库契约一致。
4. 按 03、04、05 顺序实施。

## 为什么这样合并

- 旧 03 和旧 04 合并：本地订阅镜像、详情高亮、聚合更新和每标签计数共享同一份账号状态、标签集合、事务和网络生命周期。分开开发会制造临时 API 和重复返工。
- 旧 06 和旧 07 合并：两者都不再增加主要功能，本质上是同一轮异常矩阵、迁移、性能、真机和 APK 放行工作。
- Feed 标志不合并：它有独立的检查点推进语义、列表/网格绘制和分页状态，单独验收更安全。

## 全局不可变契约

### 账号作用域

- 登录账号键使用稳定的 `ipb_member_id` 字符串。
- 游客使用固定哨兵值 `guest`，数据库键禁止使用 `NULL`。
- 网络任务启动时捕获账号键；提交数据库或更新 UI 前再次比较。
- 退出、切换账号后不得读取或显示上一账号的内存快照。

### 标签规范化

- 匹配依据始终是服务器原始完整 `namespace:value`。
- 仅做 Unicode/Locale 稳定的小写化、首尾空白移除和连续空白折叠。
- 翻译名称只用于显示，不得用于数据库键、查询或匹配。
- 规范化规则必须集中在一个纯函数中并有单测。

### 查询签名

- 签名由独立的 `QuerySignatureFactory` 产生，不由 UI 标题或最终 URL 字符串猜测。
- 包含模式、分类、实际发送的有效关键词、高级选项、页数范围和最低评分。
- 不包含分页游标或页码。
- 中文开关只有在实际改变有效关键词时才改变签名；用户明确指定其他语言时，开关变化不应产生不同签名。
- 签名采用稳定字段顺序和明确版本前缀，例如 `v1|mode=...`。

### 生命周期与任务

- 网络任务携带 `accountKey + querySignature + generation` 三项上下文。
- 旋转等配置变化可以保留进程级任务，但任务不得持有 Activity、Fragment/Scene 或 View。
- 真正离开页面只解除 UI 订阅；是否取消共享任务按产品语义决定。
- 退出账号、切换账号或有效查询变化必须取消/作废旧任务。
- 所有数据库写入和 UI 回调都要在提交前校验上下文。

### 数据安全

- schema 迁移只允许增量 `CREATE TABLE/INDEX IF NOT EXISTS`，不得删除或重建旧表。
- 新复合唯一键中的所有业务列均为 `NOT NULL`，可选值使用空串或显式哨兵。
- 迁移、镜像替换以及“检查点+计数”必须使用事务。
- 导入失败保留原数据库，不允许先清空再导入。

## 每阶段交付

每阶段必须提供：

1. 变更文件摘要和未触碰的用户改动说明。
2. 自动化测试命令、通过数、失败数和耗时。
3. 可安装 Debug APK 的绝对路径、字节数、易读大小和 SHA-256。
4. 自动验证清单与需要真实账号/真机验证的清单。
5. 已知限制和未验证原因。

构建前先执行：

```powershell
.\gradlew.bat app:tasks --all
```

任务存在时执行：

```powershell
.\gradlew.bat app:testAppReleaseDebugUnitTest
.\gradlew.bat app:assembleAppReleaseDebug
```

最终阶段额外执行 lint。不得通过关闭 lint、删除测试或吞掉异常获得绿色结果。

## 阶段完成的含义

“代码完成”不等于“阶段完成”。阶段完成要求自动测试和构建通过，APK 已产出；涉及 E-Hentai/ExHentai 服务端语义的阶段还必须列出真实账号验证结果，无法代测的项目必须明确交给项目所有者。

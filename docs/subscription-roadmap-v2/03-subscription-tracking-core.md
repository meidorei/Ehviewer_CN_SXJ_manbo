# 03 订阅追踪核心：镜像、高亮、聚合更新与计数

## 目标

本阶段合并旧 03 和旧 04：把服务器订阅标签完整镜像到本地，在详情页离线高亮，并通过 `/watched` 聚合扫描为每个订阅标签计算新增数量。合并后统一账号状态、事务、任务协调器和 UI 生命周期，避免两阶段重复改造。

## 依赖门禁

- 阶段 01 的 v8 表、`postedTimestamp`、仓库和事务 API 已完成。
- 阶段 02 的 `SearchQueryPolicy` 与 `QuerySignatureFactory` 已完成。
- 用真实账号确认：`/watched` 可分页；列表条目可通过 gdata 获得完整原始标签和发布时间；批量上限为 25。
- 若服务端验证不成立，暂停计数算法并记录实际响应，不得用猜测替代。

## A. 服务器标签镜像

- `UserTagList` 是权威来源；本地只用于快速读取和离线显示。
- 完整列表成功解析后，在单事务中替换当前账号镜像。
- 请求失败、取消、登录页 HTML、异常空页、账号变化时保留旧镜像。
- 合法的空订阅列表可以清空当前账号镜像；必须与解析失败明确区分。
- 保存 watched、hidden、服务端 ID、颜色、权重、首次发现时间和同步时间。
- 每进程会话最多自动同步一次，仅在首次打开订阅管理页触发。
- 手动更新和订阅/取消/排除成功后可执行受控同步。
- 不在应用启动、详情绑定、RecyclerView 绑定或后台定时任务中联网。

## B. 内存快照与详情高亮

- 当前账号 subscribed 原始标签一次性加载为不可变 `HashSet<String>`。
- 详情每个标签只做 O(1) 内存查询；禁止逐标签 SQL 和网络请求。
- 缓存加载完成后发布包含 `accountKey + snapshotVersion` 的事件。
- 详情页只响应当前账号和当前生命周期的事件，并在销毁时解除监听。
- watched 标签使用主题资源定义的淡黄色背景、琥珀描边和可读文字。
- 普通、排除标签及标签组标题保持原样；点击、长按、换行和翻译行为不变。
- 浅色、深色、纯黑主题分别定义资源，不在代码里硬编码唯一颜色。

## C. 标签写操作

- 长按根据本地快照显示订阅、取消订阅或排除操作。
- 只有服务端明确成功后更新本地镜像和 UI。
- 成功响应含完整 `UserTagList` 时直接事务替换；只含成功状态时再执行一次完整同步。
- 页面销毁、账号变化或 generation 变化后丢弃旧回调。
- 新订阅标签以当前聚合检查点为基线，计数初始化为 `EXACT 0`，不追溯订阅前历史。

## D. 聚合更新算法

### 任务上下文

捕获不可变的 `accountKey + effectiveQuery + querySignature + generation`。全局协调器同时只运行一个任务；它只持有应用级依赖和状态流，不持有页面对象。

### 准备

1. 同步完整服务器标签列表；失败即停止且不改旧数据。
2. 读取 watched 且非 hidden 的规范化标签 HashSet。
3. 读取 `SUBSCRIPTION_AGGREGATE` 当前检查点。
4. 中文条件通过阶段 02 的统一策略生成。

### 扫描

1. 从 `/watched` 最新页开始串行扫描，最多四页。
2. 下一页只使用服务器返回链接，不猜页码。
3. 每页通过批量 gdata 补齐完整标签与 `postedTimestamp`，每批最多 25 本。
4. 同页和跨页按 GID 去重，按发布时间倒序处理。
5. 时间小于旧边界停止；时间相等时用边界 GID 集合区分同秒新旧内容。
6. 每本新漫画的原始完整标签与订阅 HashSet 相交；一本漫画可以给多个标签各加一。
7. 任一必要 gdata 缺失、时间戳无效、解析异常或中途失败时整次不提交。

### 首次更新

- 无旧检查点时只记录当前最高时间及同秒全部 GID 作为基线。
- 所有当前订阅标签写入 `EXACT 0`。
- 不回溯历史。

### 状态

- 四页内找到旧边界：全部为 `EXACT`，显示 `N`/`0`。
- 达到四页仍未找到：匹配项为 `LOWER_BOUND`，显示 `N+`；未匹配项为 `UNKNOWN`，显示 `?`。
- 旧边界漫画被删除时仍可依靠时间小于边界停止。

### 原子提交

只有完整结束或正常达到四页上限才在一个事务中提交新检查点、全部计数和检查时间。提交前再次校验账号、签名和 generation。取消、切换账号/开关、网络失败和数据不完整均不推进。

## E. 更新 UI 和请求保护

- 订阅页提供手动刷新、页数/状态、每标签 `N`/`N+`/`?`、最近成功时间和不完整提示。
- 排除标签不显示计数。
- 点击标签不清零、不推进边界。
- 页面离开仅解除观察；旋转后重新订阅同一任务，不重复启动。
- 普通连接错误最多重试一次；限流、拒绝、服务不可用和异常 HTML 立即停止。
- 成功后冷却 60 秒，倒计时由时间戳派生，不让 View/Timer 泄漏。
- 退出或切换账号、有效查询改变时取消/作废任务。

## 测试

- 镜像事务替换、合法空列表和异常 HTML 的区别。
- 多账号、游客、退出重登与离线快照。
- 详情高亮不产生 SQL/网络，翻译不影响匹配。
- 服务端写成功/失败及旧回调。
- 单本一标签/多标签、跨页去重、同秒 GID。
- 首次基线、边界删除、四页 `EXACT` 和 `LOWER_BOUND/UNKNOWN`。
- 任一页/gdata 失败原子回滚。
- 重复点击、旋转、账号/开关切换和冷却。
- 单次列表页不超过四页，gdata 每批不超过 25，请求数与标签数无关。

## 完成标准

- 本地镜像、高亮、写操作、手动更新和计数形成一致闭环。
- 没有逐标签 SQL、逐标签网络、后台轮询或部分提交。
- 三主题、离线、多账号及生命周期验收通过。
- 提供真实账号请求日志摘要，证明串行且最多四页。
- 测试、构建和 `artifacts/step-03-subscription-core.apk` 已交付。

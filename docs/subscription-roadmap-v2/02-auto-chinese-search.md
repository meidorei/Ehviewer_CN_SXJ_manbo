# 02 全局自动加入中文搜索

## 状态与目标

当前工作区已存在本阶段实现雏形。本阶段应以“审计、补齐、测试、交付”为主，不因阶段 01 尚未完成而回退。

加入默认关闭的全局开关。开启后，仅对兼容服务端文本搜索的请求追加 `l:chinese`；搜索框、书签、分享和导入数据始终保留用户原文。

## 独立性

- 不依赖数据库 v8，可以先完成。
- 必须导出稳定的“有效查询”和查询签名输入，供阶段 01、03、04 使用。
- 后续阶段不得另行拼接中文条件。

## 纯逻辑组件

建立 `SearchQueryPolicy`，输入原始关键词、列表模式和开关状态，输出：

- `originalQuery`：用户原文，只用于 UI/保存。
- `effectiveQuery`：实际发送给服务器的关键词。
- `chineseActuallyApplied`：本策略是否真正追加中文条件。

规则：

1. 开关关闭或入口不兼容：有效查询等于原文。
2. 空关键词：有效查询为 `l:chinese`。
3. 非空关键词：仅移除末尾空白，追加一个空格和 `l:chinese`。
4. 按完整 token、大小写不敏感识别 `l:`、`lang:`、`language:`。
5. 用户已明确指定任何语言条件时用户优先，不追加中文。
6. 已有中文条件时不重复。
7. 不用简单 `contains("l:")`，避免普通文本误命中。
8. 是否将否定语言条件视为“明确语言条件”必须由测试固定；推荐用户任何显式语言表达都优先，避免改变原意。

`QuerySignatureFactory` 接受 `effectiveQuery` 和 `chineseActuallyApplied`，而不是再次执行字符串拼接。

## 兼容入口

必须覆盖：

- 主页及普通/高级搜索。
- `/watched` 订阅聚合。
- 单标签、上传者、多标签过滤。
- 搜索书签。
- 服务器收藏搜索。
- 阶段 03 的手动订阅更新。

不覆盖热门、排行、图片搜索、本地收藏、历史和下载。

标签和上传者专用 URL 在开关关闭时保持原行为；开关开启时转换成语义等价、正确转义的 `f_search`。该等价性必须用真实服务端结果验证。

## UI 与请求切换

- 设置默认 `false`，立即持久化。
- 左侧抽屉主题按钮上方放置真实可访问的开关控件，提供可读文案和状态。
- 初始化控件时先设置状态再绑定监听，避免 Activity 重建触发刷新。
- 切换后关闭抽屉。
- 当前是兼容列表时，将 generation 加一、取消旧 Call、清空旧分页并刷新第一页。
- 详情、下载、历史和本地收藏不跳页，只影响后续请求。
- 旧回调必须通过查询签名和 generation 校验，不能污染新列表。

## 对当前实现的审计清单

- 是否把策略从 `ListUrlBuilder` 中可测试地抽离，避免 UI/URL 构造器同时理解 token 语法。
- 是否覆盖服务器收藏搜索，而不仅是 `GalleryListScene`。
- 标签、上传者含空格、引号、美元符和特殊字符时是否正确。
- 抽屉项是否为预期的 `SwitchCompat`/可访问开关，而不只是 checkable menu item。
- 切换时是否真正取消旧网络请求，而不是只调用刷新。
- 是否有稳定查询签名 API。
- 页面旋转是否不会重复监听和重复刷新。

## 测试

- 空词、普通词、末尾空白。
- `l:chinese` 大小写变体和去重。
- `l:japanese`、`language:english`、`lang:korean`。
- 看似包含 `l:` 但不是完整 token 的文本。
- 否定/排除语言条件的固定语义。
- 标签和上传者转换、引号与转义。
- 开关开/关签名；相同有效查询得到相同签名。
- `QuickSearch.keyword`、输入框、分享数据不变。
- 所有兼容入口和明确不兼容入口的参数化测试。
- 旧请求晚返回不会覆盖新状态。

## 完成标准

- 所有兼容入口只通过统一策略产生有效查询。
- 用户保存和看到的仍是原文。
- 查询签名契约可供后续阶段调用。
- 真实服务器验证标签、上传者、`/watched` 和服务器收藏语义。
- 测试、构建和 `artifacts/step-02-auto-chinese-search.apk` 已交付。

# 04 Feed 检查点与“上次更新到这里”

## 目标

在主页、订阅聚合、单订阅标签和搜索书签显示“上次更新到这里”。标志代表对应入口上一次成功刷新/更新前的内容边界，不代表阅读进度。

## 依赖

- 阶段 01 的检查点表、时间戳、GID 集合和事务 API。
- 阶段 02 的有效查询与稳定签名。
- 阶段 03 的订阅聚合更新事务及旧/新边界。

## 不可变入口上下文

列表 Scene 接收显式 `FeedSourceContext`，禁止根据标题推断：

- `HOME`：固定 source key；按产品决定游客与登录账号是否共享，默认按账号隔离。
- `SUBSCRIPTION_AGGREGATE`：当前账号。
- `SUBSCRIPTION_TAG`：不单独持久化检查点，携带标签原始名、进入时的聚合旧边界和查询签名。
- `QUICK_SEARCH`：`QuickSearch.id` 与有效查询签名。
- `TEMP_SEARCH`：不持久化、不显示。

上下文进入 Scene 后不可变；切换查询应创建新上下文并移除旧 Decoration。

## 推进语义

### 主页

- 第一次成功加载第一页时，先读取旧边界供本次绘制，再提取最高有效时间和同秒全部 GID。
- 在后台事务推进检查点；当前页面继续使用进入时旧边界，不因写入立即跳线。
- 失败、异常空页或没有有效时间戳时不推进。

### 订阅聚合/单标签

- 只有阶段 03 手动更新完整原子提交时推进聚合边界。
- 聚合页使用该次更新前的 `PREVIOUS_*` 绘制。
- 单标签只使用进入时显式传入的聚合旧边界；打开列表、分页、滚动和详情均不推进。

### 搜索书签

- 点击书签并成功加载第一页时，先读取旧边界用于绘制，再推进新边界。
- 编辑后因查询签名变化自然隔离。
- 删除书签时在后台删除该账号、该 ID 的全部签名检查点。

## 边界判断

- 新内容：时间大于边界，或时间相等且 GID 不在边界集合。
- 第一本旧内容：时间小于边界，或时间相等且 GID 在边界集合。
- 边界漫画被删除时依靠时间小于边界定位。
- `postedTimestamp == 0` 不参与推进、比较或命中。
- 若列表排序不是发布时间倒序，禁用标志并记录原因，不能画出误导性边界。

## RecyclerView 绘制

- 使用 `ItemDecoration`，禁止插入虚假 `GalleryInfo`。
- Decoration 只读取当前适配器快照与可见 View，不访问数据库或网络。
- `getItemOffsets` 在边界所在项/网格整行上方只预留一次高度。
- 列表横跨内容宽度；网格横跨边界项所在整行，不能切断卡片。
- 考虑 RecyclerView padding、RTL、主题颜色和字体缩放。
- 边界在后续页时，直到该页加载后才出现。
- 刷新、分页追加、布局切换和上下文变化后调用 `invalidateItemDecorations()`。
- Scene 销毁或切换查询时显式移除 Decoration，避免残留。

## 测试

- 大于/等于/小于边界及同秒 GID 集合。
- 边界漫画删除、无效时间戳、非时间排序。
- 首页无边界、后续页出现。
- 中文实际生效与未生效时的签名隔离。
- 临时搜索无检查点；点击、滚动、详情和分页不推进。
- 删除书签清理全部签名。
- 列表/多列网格、布局切换、字体缩放和查询切换。

## 完成标准

- 四类入口按各自语义显示，普通搜索不显示。
- 适配器数据、稳定 ID、点击位置和分页行为未被修改。
- 边界只由规定的成功事件推进，当前页面使用推进前快照。
- 测试、构建和 `artifacts/step-04-feed-marker.apk` 已交付。

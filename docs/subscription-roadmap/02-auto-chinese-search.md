# 02 全局“自动加入中文搜索”

## 阶段目标

加入一个默认关闭的全局开关。开启后，所有兼容服务器文本搜索的请求在真正构建 URL 时自动加入 `l:chinese`，但搜索框、搜索书签和分享数据继续保留用户原文。本阶段完成后单独交付 APK，先验证搜索覆盖面和语法，不等待订阅计数功能。

## 依赖

- 依赖阶段 01 的稳定构建和查询签名基础。
- 不依赖详情标签高亮，可与阶段 03 并行。

## 设置与左侧抽屉 UI

- 在 `Settings` 增加布尔配置，默认 `false`。
- 在左侧抽屉底部控制区、主题按钮上方增加 `SwitchCompat`，文案为“自动加入中文搜索”。
- 初始化时读取设置，切换时立即持久化。
- 切换后关闭抽屉；若当前页面是兼容的服务器漫画列表，取消旧请求并按新策略刷新第一页。
- 当前为详情、历史、下载、本地收藏等页面时不强制跳页，只影响之后的请求。
- 旋转屏幕或 Activity 重建不得重复触发监听器和多次刷新。

## 统一查询策略

新增纯逻辑组件 `SearchQueryPolicy`，所有兼容入口必须在最终生成 URL 前调用，禁止各页面自行拼字符串。

规则：

1. 开关关闭：原样返回。
2. 空关键词：有效关键词为 `l:chinese`。
3. 非空关键词：去除末尾多余空白后追加一个空格和 `l:chinese`。
4. 大小写不敏感、按完整 token 检测 `l:` 和 `language:`。
5. 用户已写任意语言条件时，用户条件优先，不追加中文。
6. 已有 `l:chinese` 时不重复。
7. 不修改搜索框文本、`QuickSearch.keyword`、分享 URL 或导入的原始数据。
8. 查询签名使用“实际发送的有效查询”，不使用搜索框原文。

不要用简单 `contains("l:")` 判断，因为标题或普通词的一部分可能误命中；应使用可测试的 token 解析或边界正则。

## 覆盖入口

必须覆盖：

- 主页。
- 普通搜索与高级搜索。
- 左侧订阅聚合页 `/watched`。
- 后续的订阅手动更新。
- 单个订阅标签搜索。
- 右侧搜索书签。
- 详情标签点击后的搜索。
- 上传者搜索。
- 多标签过滤搜索。
- 服务器收藏搜索。

明确不覆盖：热门、排行榜、图片搜索、本地收藏、历史、下载。这些入口没有等价文本搜索参数。

## 标签与上传者专用 URL

- 开关关闭时保持原专用 URL 行为。
- 开启时转换成等价普通 `f_search` 表达式，再加入语言条件。
- 标签始终使用服务器原始完整名称；含空格、引号或特殊字符时正确转义和引用。
- 上传者同样转成准确的 uploader 搜索表达式。
- 用真实服务器结果对比开关关闭时的专用 URL，确认语义一致；若服务器语法有差异，只调整转换器，不让 UI 模块理解语法细节。

## 请求并发与状态

- 切换开关时给当前查询生成新签名/代次。
- 旧请求即使稍后返回，也不得覆盖新开关状态下的列表。
- 书签保存的是原始关键词；同一书签在开关开/关时产生不同有效签名。

## 自动化测试

- 空词、普通词、末尾空格。
- `l:chinese` 大小写变体和重复条件。
- `l:japanese`、`language:english` 等显式语言优先。
- 看似包含 `l:` 但不是语言 token 的文本不误判。
- 含空格/引号的标签转换。
- 上传者转换。
- 开关开/关签名不同，相同有效查询签名稳定。
- `QuickSearch.keyword` 和输入框模型不被修改。

## 手机验收

1. 开关关闭，记录主页和若干搜索结果。
2. 开启后主页自动刷新，结果只包含中文语言条件。
3. 搜索普通词，确认输入框没有显示自动追加文本。
4. 输入 `l:japanese`，确认没有再追加中文。
5. 测试标签、上传者、订阅聚合、搜索书签和服务器收藏。
6. 关闭开关，确认恢复原查询且没有旧请求回写。
7. 重启应用，确认开关状态持久化。

## 完成标准

- 所有兼容入口走统一策略。
- 用户原始搜索和书签没有被改写。
- 显式语言条件优先。
- 旧回调不会污染新查询。
- 已交付可安装的阶段 APK 和真实服务器冒烟测试记录。

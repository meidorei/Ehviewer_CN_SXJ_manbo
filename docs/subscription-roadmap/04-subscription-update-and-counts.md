# 04 安全聚合更新、标签新增计数与订阅页 UI

## 阶段目标

在右侧订阅标签页加入手动更新，为每个“订阅中”标签显示 `N`、`N+` 或 `?`。更新只扫描服务器 `/watched` 聚合列表，网络请求量与订阅标签数量无关。此阶段完成后即可在手机上完整测试“手动刷新—看到每标签新增数”。

## 依赖

- 阶段 01：检查点、计数表和原始发布时间。
- 阶段 02：订阅更新必须使用同一个有效查询策略和签名。
- 阶段 03：本地订阅集合、账号隔离和服务器镜像。

## 明确禁止的方案

- 禁止为每个订阅标签分别搜索。
- 禁止后台轮询、WorkManager、定时刷新和应用启动自动扫描。
- 禁止并发请求四个列表页。
- 禁止网络中断后提交部分计数。
- 禁止以设备当前时间作为内容边界。

## 更新算法

### 1. 准备阶段

- 捕获当前账号键、中文开关状态、有效查询和查询签名，形成不可变任务上下文。
- 同步一次完整服务器标签列表；失败时保留本地镜像和旧计数，更新任务停止。
- 获取当前账号“订阅中且未排除”的原始规范化标签集合。
- 读取 `SUBSCRIPTION_AGGREGATE` 当前检查点。
- 全局更新协调器确保同时只有一个任务；重复点击只复用状态或提示正在更新。

### 2. 聚合扫描

1. 从 `/watched` 最新页开始串行请求。
2. 使用项目现有列表解析，并通过批量 `gdata` 补齐每本漫画的完整标签和 `postedTimestamp`。
3. `gdata` 每批最多 25 本，沿用现有客户端限制。
4. 按时间倒序处理漫画；同页或跨页按 GID 去重。
5. 时间早于旧边界时停止；时间等于边界时用旧边界 GID 集合区分旧内容和同秒新内容。
6. 对每本新漫画，将其完整原始标签与本地订阅 HashSet 相交；一本漫画可给多个标签各加一，同一标签内同一 GID 只计一次。
7. 最多扫描 4 页。下一页 URL 使用服务器返回链接，不自行猜页码。

如果列表 HTML 本身缺少完整标签，必须调用 `gdata`。只要本页任何必要 `gdata` 项缺失或时间戳无效，本次任务就不提交，以免少算后错误推进边界。

### 3. 首次更新

- 没有旧检查点时，仅把当前最新时间及该时间下所有 GID 保存为基线。
- 所有已有订阅标签初始化为精确 `0`。
- 不回溯订阅前的历史内容。

### 4. 精确与不完整状态

四页内找到旧边界：

- 所有订阅标签状态为 `EXACT`。
- 匹配到的显示 `N`，未匹配显示 `0`。

四页内未找到旧边界：

- 已匹配标签为 `LOWER_BOUND`，显示 `N+`。
- 扫描窗口内未匹配的标签为 `UNKNOWN`，显示 `?`，不能显示 `0`。
- 页面显示“仍有更早内容未统计”。

### 5. 原子提交

- 只有完整扫描结束或正常达到四页上限才提交。
- 在同一数据库事务中写入：新订阅聚合检查点、全部标签计数、检查时间。
- 网络失败、取消、账号切换、中文开关切换、页面销毁或解析不完整时均不推进边界。
- 提交前再次比较账号键和任务代次，防止旧任务覆盖新状态。

## 请求保护

- 同时只允许一个订阅更新任务。
- 页面请求串行；普通连接错误最多指数/固定延迟重试一次。
- HTTP 限流、拒绝、服务器不可用或异常页面立即停止，不连续重试。
- 完整结束后进入 60 秒冷却；失败是否进入短冷却可按错误类型决定，但不得形成点击风暴。
- 冷却期间刷新按钮禁用并显示剩余秒数，重复点击不创建任务。
- 页面关闭、退出登录、切换账号或中文开关切换时取消网络 Call。
- 旋转屏幕由进程级协调器保留任务，重建页面只重新订阅进度，不重复启动。

## 订阅标签页 UI

- 工具栏加入手动刷新按钮。
- 刷新期间显示旋转/脉冲进度、当前扫描页或简短状态。
- 每个订阅标签右侧显示 `N`、`N+` 或 `?`。
- 排除标签继续显示在管理列表中，但不显示新增角标。
- 显示最近一次成功更新时间。
- 达到四页上限时显示统计不完整提示。
- 冷却期间显示剩余时间；离开页面后不保留引用页面 View 的计时器。
- 点击标签继续执行普通标签搜索，不清零角标，也不推进更新边界。

## 新订阅标签初始化

- 新订阅或首次从服务器发现的订阅标签以当前聚合检查点为基线。
- 初始计数写为精确 `0`。
- 不把订阅之前的历史内容计为新增。
- 删除/排除标签时清理或停止展示相应计数；不得影响其他标签。

## 网络请求量预期

- 常见：1 个 `/watched` 列表请求 + 1 个批量标签请求。
- 极端：最多 4 个列表请求，以及每页必要的批量请求。
- 请求量只跟扫描页内容数量有关，与订阅了 5 个还是 500 个标签无关。

## 自动化测试

- 单本漫画匹配一个/多个订阅标签。
- 标签内、跨页 GID 去重。
- 首次更新全为 `0`。
- 同时间戳边界 GID。
- 旧边界漫画被服务器删除后仍按时间停止。
- 四页内找到边界得到 `N/0`。
- 四页未找到得到 `N+/?`。
- 第 2/3/4 页失败不提交任何新状态。
- 账号、签名或任务代次变化后旧回调被丢弃。
- 重复点击只产生一个任务，冷却生效。
- 单次列表页数绝不超过 4。

## 手机验收

1. 首次更新：所有订阅标签显示 `0`。
2. 服务器出现新漫画后再更新：相关标签增加；一本多标签漫画分别计入。
3. 连续点击刷新：无并发请求，看到冷却倒计时。
4. 更新中断网：旧计数和边界不变。
5. 更新中旋转、关闭页面、切换账号和切换中文开关。
6. 验证排除标签无角标。
7. 用代理日志或客户端日志确认没有逐标签请求，最多四页。

## 完成标准

- 手动更新、计数、状态提示和冷却可用。
- 请求数与标签数无关，最多扫描四页。
- 所有更新结果原子提交。
- 中断和过期回调不推进边界。
- 已交付阶段 APK、SHA-256 与真实账号请求日志摘要。

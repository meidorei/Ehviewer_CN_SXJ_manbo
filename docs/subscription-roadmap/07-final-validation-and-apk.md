# 07 全量测试、性能审计与最终 APK 交付

## 阶段目标

冻结功能范围，完成全量回归、迁移验证、lint、构建和可安装 Debug APK 交付。只有所有硬性完成标准满足后才宣布完成；真实服务器无法自动化的部分必须列为明确的冒烟验证，而不是省略。

## 依赖

- 阶段 01 至 06 全部完成并各自产出过可安装 APK。
- 阶段 06 候选版手机测试发现的问题已经修复或进入书面已知问题清单。

## 代码冻结审查

- 审查 `git diff`，确认没有无关格式化、生成物、签名密码、cookie、日志隐私数据或意外大文件。
- 检查数据库 schema 版本、全新建库和升级路径一致。
- 检查所有新增字符串、主题资源和无障碍文案。
- 检查取消订阅等服务器写操作仍以服务端成功为准。
- 检查没有重新生成并覆盖现有定制 DAO。
- 检查所有线程、回调和监听器的生命周期。

## 单元测试矩阵

至少覆盖：

### 标签与账号

- 原始完整标签规范化。
- 翻译显示不影响匹配。
- 订阅/排除状态。
- 多账号和游客隔离。
- 新订阅以当前边界初始化为 `0`。

### 更新算法

- 一对一、一对多标签匹配。
- 标签内和跨页 GID 去重。
- 首次基线。
- 同秒边界 GID。
- 边界漫画删除。
- `EXACT` 的 `N/0`。
- `LOWER_BOUND` 的 `N+`。
- 未扫描完整时的 `?`。
- 四页硬上限。
- 任一页或 gdata 不完整时原子回滚。

### 搜索策略

- 中文追加、去重、显式其他语言优先。
- 空查询。
- 标签/上传者转换与转义。
- 所有兼容入口使用实际查询。
- 不兼容入口不受影响。
- 搜索框、书签和分享原文不被修改。
- 查询签名稳定且开关隔离。

### 检查点

- 主页、订阅、标签和书签来源。
- 只在规定成功事件推进。
- 时间和 GID 联合比较。
- 普通搜索无边界。
- 删除书签清理。

## 数据库测试

- 创建全新 v8 数据库。
- 从真实结构 v7 fixture 升级到 v8。
- 升级前后原表行数、主键和关键字段一致。
- 三张新表可读写、导出和导入。
- v7 备份可导入，v8 备份可往返。
- 迁移重复执行不破坏数据。
- 模拟失败不会清空原数据库。

## UI 与生命周期测试

- 三种主题标签高亮和文字可读。
- 长按成功即时更新，失败不假成功。
- 订阅页角标、刷新进度、四页提示和冷却。
- 列表/网格的边界线，包含后续页。
- 页面、账号、开关快速切换无旧回调覆盖。
- 旋转不重复任务，销毁后不泄漏。
- 离线缓存显示与错误提示正确。

## 必须执行的构建命令

先确认任务：

```powershell
.\gradlew.bat app:tasks --all
```

再依次执行：

```powershell
.\gradlew.bat app:testAppReleaseDebugUnitTest
.\gradlew.bat app:lintAppReleaseDebug
.\gradlew.bat app:assembleAppReleaseDebug
```

如果项目实际任务名不同，只替换为 `tasks --all` 显示的等价任务，并在交付报告写明。不得通过关闭 lint 或删除失败测试来获得绿色构建。

## 最终真实账号冒烟测试

- 获取服务器订阅列表并缓存。
- 详情订阅标签即时高亮；断网后仍显示缓存。
- 订阅/取消/排除成功与失败路径。
- 首次手动更新为 `0`，后续新增得到合理计数。
- 请求日志确认不按标签请求、最多四页、冷却有效。
- 中文开关覆盖主页、普通搜索、订阅、标签、上传者、书签和服务器收藏。
- 主页、聚合、单标签和书签边界标志正确。
- 账号切换和登录过期不串数据。

## APK 与校验值

构建变体：

- Flavor：`appRelease`。
- Build type：`debug`。
- Application ID：`com.xjs.ehviewer.debug`。
- 当前版本：`versionCode 111`、`versionName 2.0.2.2`；若开发期间版本变化，以构建文件为准并报告。
- 使用 Android 调试签名，不修改正式签名配置。

预期原始输出：

```text
app/build/outputs/apk/appRelease/debug/app-appRelease-debug.apk
```

最终复制与校验：

```powershell
Copy-Item app\build\outputs\apk\appRelease\debug\app-appRelease-debug.apk artifacts\ehviewer-subscription-final-debug.apk
Get-Item artifacts\ehviewer-subscription-final-debug.apk | Select-Object FullName,Length,LastWriteTime
Get-FileHash artifacts\ehviewer-subscription-final-debug.apk -Algorithm SHA256
```

## 最终交付报告格式

必须提供：

- 文件字节数和易读大小。
- SHA-256。
- 三条 Gradle 命令的结果和耗时。

- v7→v8 迁移验证摘要。
- 手机/真实账号已验证项目。
- 仍无法自动验证的项目及原因。
- 修改文件按“数据库、网络、UI、测试”分类摘要。
- 已知限制，例如 Debug 包不能直接读取正式版数据目录。

## 最终完成标准

- 所有计划功能已编码且阶段回归通过。
- 新增单元测试和 v7→v8 迁移测试通过。
- lint、资源合并、编译、APK 签名成功。
- 没有主线程逐标签数据库查询。
- 没有逐订阅标签网络请求。
- 单次更新最多四页，无后台轮询。
- 中文开关不修改用户原始搜索或书签。
- 过期网络回调不能覆盖当前账号/查询。
- 最终 APK、校验值、验证报告和已知限制均已交付。

# 05 Feed 检查点与“上次更新到这里”标志

## 阶段目标

在主页、订阅聚合浏览、单个订阅标签搜索和搜索书签结果中显示“上次更新到这里”。标志代表对应入口上一次成功更新/加载首屏时的发布时间边界，不代表用户点开或真正阅读到了哪一页。

## 依赖

- 阶段 01 的 `FEED_CHECKPOINT`、原始时间戳和 GID 集合。
- 阶段 02 的有效查询签名。
- 阶段 04 的订阅聚合手动更新边界。

## 语义定义

- 文案固定为“上次更新到这里”，避免误称“上次阅读到这里”。
- 主页：上一次主页首屏成功刷新时的边界。
- 订阅聚合：上一次手动更新成功前的旧边界。
- 单订阅标签：进入标签搜索时引用订阅聚合的旧边界，并结合当前有效查询签名判断是否适用。
- 搜索书签：上一次点击该书签并成功加载首屏时的边界。
- 点击漫画、进入详情、滚动列表、翻页均不推进边界。
- 普通临时搜索默认不保存检查点，也不显示标志。

## 查询来源识别

为列表 Scene 建立明确的不可变入口上下文，而不是根据标题猜测：

- `HOME`：固定 source key。
- `SUBSCRIPTION_AGGREGATE`：当前账号键。
- `SUBSCRIPTION_TAG`：携带标签原始名、聚合旧边界及有效签名。
- `QUICK_SEARCH`：携带 `QuickSearch.id`。
- `TEMP_SEARCH`：无检查点。

从右侧抽屉点击标签或书签时必须显式传递上下文。切换查询、返回、恢复状态时重新计算，禁止沿用上一个列表的 Decoration。

## 边界推进规则

### 主页

- 成功加载第一页后，从有效 `postedTimestamp > 0` 的最新项提取最高时间。
- 收集该最高时间下当前结果中的全部 GID。
- 事务推进主页检查点。
- 请求失败、空异常页或只有无效时间戳时不推进。

### 订阅聚合与单标签

- 只有阶段 04 的手动更新完整提交时推进聚合边界。
- 打开聚合列表、点击单标签、加载列表或打开漫画均不推进。
- 单标签页面只读取进入时传入的旧边界用于绘制。

### 搜索书签

- 点击书签并成功加载第一页后推进该 `QuickSearch.id` 的检查点。
- 先读取旧检查点用于本次绘制，再提交新边界；不得因为写入后立刻把标志跳到最新位置。
- 删除书签时删除其全部签名检查点。
- 编辑书签内容后因有效查询签名变化，自然使用独立边界。

## 同时间戳和删除内容

- 判断“新内容”：时间大于边界，或时间等于边界且 GID 不在边界集合。
- 判断第一本“旧内容”：时间小于边界，或时间等于边界且 GID 在集合中。
- 即使原先边界中的某一本被服务器删除，仍可依靠时间找到位置。
- 旧缓存 `postedTimestamp == 0` 不参与推进，也不能被当作边界命中。

## `RecyclerView.ItemDecoration`

禁止插入虚假的 `GalleryInfo`，避免破坏适配器位置、点击、分页、选择和稳定 ID。

实现要求：

- Decoration 只检查当前已加载数据和可见 View。
- 在第一本属于旧边界的漫画上方通过 `getItemOffsets` 预留空间。
- `onDraw`/`onDrawOver` 绘制横线和居中文字。
- 列表模式横跨可用宽度。
- 网格模式定位边界所在整行，在该行上方横跨整行，只预留一次高度。
- 考虑左右 padding、RTL（如项目支持）、状态栏/抽屉宽度和主题文字颜色。
- 边界在后续页时，前面页面不显示；翻到包含边界的页后再显示。
- 当前页无边界或查询无上下文时移除 Decoration。
- 数据刷新、布局模式切换、分页追加后调用 `invalidateItemDecorations()`。

## 查询签名隔离

签名必须包含规范化后的：

- 搜索模式、分类、有效关键词。
- 高级搜索选项。
- 页数与最低评分条件。
- 中文条件是否实际生效。

签名不包含分页页码。中文开/关、显式其他语言、不同书签内容不得共用边界。

## 自动化测试

- 大于、等于、小于边界的分类。
- 同时间戳 GID 集合。
- 边界漫画被删除。
- 首页没有边界、后续页出现边界。
- 查询签名开关隔离。
- 临时搜索不创建检查点。
- 点击/滚动/打开详情不推进。
- 删除书签清理检查点。
- 切换查询后 Decoration 不残留。

## UI/手机验收

1. 主页刷新两次，第二次能在正确漫画上方看到标志。
2. 手动更新订阅后进入聚合页和某个订阅标签，标志代表更新前边界。
3. 点击搜索书签两次，验证边界推进；普通搜索不显示。
4. 测试列表和多列网格，横线不切断单个卡片。
5. 翻页直到边界出现。
6. 切换中文开关，旧签名标志不复用。
7. 打开漫画再返回，标志不移动。

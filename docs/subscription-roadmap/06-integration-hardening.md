# 06 生命周期、账号隔离、导入导出与异常加固

## 阶段目标

不再增加主要产品功能，集中修复跨页面、账号切换、配置变化、数据库备份和异常响应的问题。完成后交付“候选版”APK，用于较长时间真实账号测试。

## 依赖

- 阶段 01 至 05 全部完成。

## 生命周期与并发模型

- 所有网络任务携带账号键、查询签名和递增任务代次。
- 回调提交数据库或更新 UI 前再次核对三项上下文。
- Scene/View 销毁后解除 EventBus、LiveData、回调和倒计时观察者。
- 进程级更新协调器可以跨旋转保留任务，但不得持有 Activity、View 或短生命周期 Context。
- 页面关闭只取消该页面独占的任务；订阅更新按产品约定取消时必须确保不提交部分结果。
- 退出登录、切换账号和中文开关改变立即取消不再适用的请求。
- 主线程只做轻量集合查找和 UI 绘制；SQLite、解析、匹配、哈希和大集合构造放后台线程。

## 账号隔离审计

- 所有订阅镜像、更新计数、订阅检查点都必须带 `accountKey`。
- 主页若采用公共作用域，需要文档化；如果结果受登录域、中文条件或账号偏好影响，应使用当前账号作用域。
- 登录态发生变化时刷新内存 HashSet，禁止短暂显示旧账号黄色标签。
- 游客访问订阅管理时明确提示登录，不发无效更新请求。
- 本地有旧缓存但离线时允许高亮，并在订阅页标明缓存时间；不能伪装成刚同步成功。

## 数据库导入导出

- 现有数据库导出必须包含三张新表。
- 导入到 v8 时恢复镜像、检查点和计数，并在事务完成后重建内存缓存。
- 导入旧 v7 备份时允许缺少三张表，导入结束后创建空表。
- 导入不同账号数据时保留账号键，不合并成当前账号。
- 若现有导出机制复制整个 SQLite 数据库，验证 WAL/checkpoint 处理；若逐表复制，添加明确列映射。
- 导入失败保留原数据库，提供错误日志，禁止静默清空。

## 错误矩阵

逐项实现并验证：

- 未登录：管理、更新和服务器写操作提示登录。
- 普通断网：保留旧镜像、计数和检查点。
- 限流/拒绝/服务器不可用：立即停止，不连续重试。
- 异常 HTML/登录页重定向：视为解析失败，不能把它当空订阅列表。
- 服务器返回合法空订阅：清空当前账号镜像和相应计数。
- `gdata` 部分缺失：整次更新不提交。
- 漫画被删除：时间边界仍能工作。
- 同秒多本：GID 集合正确。
- 删除搜索书签：清理检查点。
- 中文开关改变：新旧签名隔离并取消旧请求。
- 快速切换页面：旧回调不覆盖当前 UI。

## 性能审计

使用代码审查、日志和 Android Profiler 验证：

- 详情标签绑定没有网络和逐标签 SQL。
- 内存匹配为 HashSet O(1)。
- 完整镜像替换为单事务。
- 更新期间列表请求串行，最多四页。
- gdata 保持每批最多 25 本。
- Decoration 只处理可见 View/当前数据，不在每帧扫描整个数据库。
- SearchQueryPolicy 每个请求只运行一次。
- 没有后台轮询、WorkManager 或隐藏定时网络任务。
- 页面销毁后无明显 Activity/Scene 泄漏。

## 可观测性

- 为更新任务记录结构化调试日志：账号键应脱敏、查询签名、页数、条目数、停止原因、结果状态和耗时。
- 不记录 cookie、API key 或完整隐私搜索历史。
- 错误提示区分：网络失败、登录失效、服务器限流、解析失败、统计达到上限。
- Release 可降低详细日志，Debug 保留足够诊断信息。

## 自动化与压力测试

- 旋转屏幕多次只运行一个更新任务。
- 任务中途切账号/开关，不提交旧结果。
- 500 个本地订阅标签、每页常规漫画量下匹配耗时可接受。
- 重复打开关闭详情无监听器累积。
- v7 备份导入 v8、v8 完整导出再导入。
- 合法空列表与异常登录 HTML 能区分。
- 数据库事务故障模拟后旧数据完整。

## 手机候选版验收

- 连续使用主页、详情、订阅更新、标签搜索和书签至少一个完整会话。
- 前后台切换、锁屏、旋转、断网恢复、VPN/IP 切换。
- 两个账号交替登录（如具备条件）。
- 更新过程中关闭抽屉、返回主页或退出账号。
- 观察请求日志：无逐标签请求、无后台自动扫描、最多四页。
- 检查三主题、列表/网格和不同字体缩放。

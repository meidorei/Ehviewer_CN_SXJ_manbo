# 03 本地订阅镜像与详情标签高亮

## 阶段目标

把服务器订阅/排除标签完整镜像到本地，并在漫画详情页仅依赖本地缓存高亮已订阅标签。打开详情页不得新增网络请求。完成后交付 APK，重点验证同步、离线显示、账号隔离和三种主题。

## 依赖

- 依赖阶段 01 的 `SUBSCRIPTION_TAG_CACHE` 和仓库基础 API。
- 不依赖阶段 02，可与其并行；合并时以服务器原始标签名作为唯一匹配依据。

## 本地镜像规则

- 服务器 `UserTagList` 是唯一权威来源，本地缓存只用于快速读取和离线显示，不反向覆盖服务器。
- 逻辑唯一键为 `accountKey + normalizedTagName`。
- 完整列表同步成功后，在单个事务内替换当前账号缓存；请求失败、取消、异常 HTML 或账号切换时不改旧缓存。
- 保存订阅、排除、服务器标签 ID、颜色、权重、首次发现时间和同步时间。
- 不同 `ipb_member_id` 严格隔离。游客不得看到旧登录账号的标签。
- 退出登录停止使用旧账号缓存，但可保留数据库行，便于同账号重新登录后离线显示。

## 同步时机

- 每个应用进程会话最多自动同步一次。
- 首次打开右侧订阅标签页且本会话尚未同步时请求服务器。
- 用户后续执行订阅更新时同步。
- 长按标签订阅、取消订阅或排除成功，且服务器返回完整列表时立即同步。
- 不在应用启动、详情打开、RecyclerView 绑定或后台定时任务中自动请求。

## 内存读取策略

- 当前账号的已订阅标签一次性加载为不可变 `HashSet<String>`。
- 详情绑定每个标签只做 O(1) 集合查询。
- 禁止每个标签单独查 SQLite，也禁止逐标签网络请求。
- 翻译只影响文字显示；匹配始终使用服务器原始 `namespace:value`。
- 本地缓存初始化异步进行；完成后发出带账号键的状态事件，当前可见详情重新绑定。

## 详情页样式

现有 `TextView + RoundSideRectDrawable` 保持不变，只扩展状态样式：

- 普通标签：原灰色。
- 已订阅：淡黄色背景、琥珀色约 1dp 描边、文字满足对比度。
- 排除：本阶段仍用普通样式，不和订阅混淆。
- 标签组名称不高亮。
- 点击、长按、自动换行和标签翻译行为不变。

为浅色、深色、纯黑主题分别定义 `subscribedTagBackgroundColor` 与 `subscribedTagStrokeColor` 属性及颜色资源，不在 Java/Kotlin 中硬编码单一颜色。扩展 drawable 时要保持圆角、alpha 和 color filter 行为兼容。

## 长按标签操作

- 本地未订阅：显示“订阅”。
- 本地已订阅：显示“取消订阅”或项目服务器接口对应的移除操作。
- 排除状态使用独立文案。
- 只有服务器成功后才更新本地镜像与当前页面；失败时保持原颜色并提示错误。
- 若服务端成功响应含完整 `UserTagList`，直接事务性替换；若只返回成功状态，再执行一次受控完整同步。
- 页面销毁或账号切换后，旧回调不得修改当前账号 UI。

## 状态事件与生命周期

- 同步成功后发布本地事件，事件至少包含账号键和状态版本。
- 详情页与右侧订阅列表在主线程重绑。
- `onDestroyView`/`onDestroy` 必须解除监听，禁止 Activity/Scene 泄漏。
- 旋转屏幕不重复启动同步；新页面读取仓库当前快照。

## 自动化测试

- 大小写、空白规范化，以及翻译开关不影响匹配。
- 订阅与排除区分。
- 多账号/游客隔离。
- 完整替换保留未变化标签的 `FIRST_SEEN_AT`。
- 失败事务保留旧镜像。
- 同一进程自动同步门禁只触发一次。
- 服务器成功前不提前高亮。

## 手机验收

1. 登录后打开右侧订阅页，进入包含该标签的漫画详情，标签变淡黄色。
2. 切换浅色、深色和纯黑主题，检查文字和描边。
3. 长按订阅/取消，成功后当前页即时更新。
4. 断网重开详情，已有缓存仍高亮且不产生详情附加请求。
5. 切换账号，确认标签集合不会串号。
6. 快速退出详情或旋转屏幕，确认无崩溃和过期回调。

## 完成标准

- 详情标签匹配不做网络请求或逐项 SQL。
- 三种主题显示正确。
- 服务器成功后才改变本地状态。
- 离线与账号隔离符合预期。
- 已交付可安装阶段 APK、校验值和冒烟结果。
